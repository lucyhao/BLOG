JavaScript

  • Explain event delegation 解释事件代理

答：如果一堆子元素在一个父元素的里面,然后又要给这些子元素绑定事件,那么我们可以使用事件代理。也就是不需要给每一个子元素都绑定事件，而是把事件绑定到父元素上，然后通过event.target来获得具体触发的子元素。这样做的好处是，减少了冗余的事件绑定，节约了多余的事件带来的开销。
(If there are many element inside one parent, and you want to handle events on them of them - don’t bind handlers to each element.Instead, bind the single handler to their parent, and get the child from event.target.
Simplifies initialization, saves memory from extra handlers
)

＊ this的指代情况

this大多数都是由函数调用的上下文和它在哪里被调用的所决定。在全局上下文调用的时候，指代的是window；在对象方法中调用的时候，指向此的对象

(In JavaScript the value of this is determined mostly by the invocation context of function (context.function()) and where it is called.
when used in global context, it is window
When used inside object method, it bound to the immediate enclosing object.
)

  • prototypal inheritance works 原型链

每个对象都有一个prototype指向另外一个对象，然后另外的那个对象也有自己的prototype指向别的对象。这样的一个链直到null.
与面向对象语言不同的是，可以随时给对象增加属性到prototype上。
(Each object has an internal link to another object called its prototype. That prototype has its own prototype. And like this until the null. All of this is called prototype chain.
)

在闭包里面可以访问它的父作用域，甚至在父作用域已经返回的了情况下。所以闭包可以访问三个“区域”的变量，一个是它自己的，一个是包裹它的父作用域的，一个是全局的。
A closure is a function having access to the parent scope, even after the parent function has closed.
